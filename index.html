Aprenda IF, ELSE IF e ELSE em Java

Guia completo para dominar estruturas condicionais em Java do jeito mais simples possível

1. O Básico: Entendendo as Condicionais

No mundo da programação, as estruturas condicionais são como a capacidade de tomar decisões no dia a dia. Imagine que você acorda e pensa: "Se estiver sol, vou à praia. Senão, se estiver nublado, vou ao parque. Caso contrário (se chover), vou ficar em casa."

Em Java, e em outras linguagens, as estruturas if, else if e else permitem que seu programa siga diferentes caminhos lógicos dependendo de certas condições serem verdadeiras ou falsas. Elas são fundamentais para criar programas que reagem a entradas do usuário, condições do sistema ou valores de variáveis, tornando-os dinâmicos e inteligentes.

Dominar as condicionais é um passo crucial para qualquer desenvolvedor, pois elas são a espinha dorsal da lógica de negócios em quase todo software que você encontrará.

2. IF, ELSE IF e ELSE: As Ferramentas da Decisão

2.1. O que é if?

O if (se) é a estrutura condicional mais básica. Ele testa uma única condição. Se a condição for verdadeira, o bloco de código dentro do if é executado. Se for falsa, o bloco é ignorado, e o programa continua sua execução a partir da linha seguinte ao if (ou else, se houver).

Sintaxe do if:

if (condicao) {
    // Código a ser executado SOMENTE se a condicao for verdadeira
}
// O programa continua aqui, independentemente da condicao

2.2. O que é else?

O else (senão) atua como uma "alternativa" ao if. Ele sempre vem junto com um if. Se a condição do if for falsa, o bloco de código dentro do else é executado. A lógica é simples: "Se isso for verdade, faça A. Senão, faça B."

Sintaxe do if-else:

if (condicao) {
    // Código para quando a condicao for verdadeira
} else {
    // Código para quando a condicao for falsa (a alternativa)
}

2.3. O que é else if?

O else if (senão se) é usado para testar múltiplas condições em sequência. Ele sempre vem depois de um if (ou outro else if). Se a condição do if inicial for falsa, o programa passa para o primeiro else if e testa sua condição. Se for verdadeira, executa seu bloco e pula o restante. Se for falsa, vai para o próximo else if (se houver), e assim por diante.

Você pode ter quantos else ifs precisar em uma cadeia. É como um "cascateamento" de decisões.

Sintaxe do if-else if-else:

if (primeiraCondicao) {
    // Executa se a primeira condição for verdadeira
} else if (segundaCondicao) {
    // Executa se a primeira for falsa E a segunda for verdadeira
} else if (terceiraCondicao) {
    // Executa se a primeira e segunda forem falsas E a terceira for verdadeira
} else {
    // Executa se TODAS as condições anteriores forem falsas (opcional)
}
Comparação: O if é para uma decisão única. O if-else oferece dois caminhos. O if-else if-else permite múltiplos caminhos, testando um após o outro até encontrar uma condição verdadeira ou cair no else final.

3. Regras de Ouro das Condicionais em Java

Para usar if, else if e else corretamente em Java, fique atento a estas regras:

3.1. Estrutura dos Blocos com Chaves { }

Em Java, os blocos de código associados a um if, else if ou else são delimitados por chaves { }. Embora seja possível omitir as chaves para um único comando, é uma **má prática** e altamente desencorajado.

Sempre use chaves { }: Isso melhora a legibilidade do código, evita erros quando você ou outro programador adicionar mais linhas de código no futuro e define claramente o escopo do bloco.

3.2. Comparando Valores: == vs .equals()

Esta é uma das maiores fontes de erro para iniciantes em Java e crucial para entender.

  • == (Operador de Comparação):
    • Para tipos primitivos (int, double, boolean, char), ele compara os valores.
    • Para objetos (como String), ele compara as referências de memória, ou seja, verifica se as duas variáveis apontam para o *mesmo* objeto na memória. Raramente é o que você quer ao comparar Strings.
  • .equals() (Método de Objeto):
    • É um método disponível para todos os objetos (incluindo String) que compara o conteúdo dos objetos.
    • Para Strings, .equals() verifica se os caracteres de uma String são idênticos aos da outra.

A regra é: Sempre use .equals() para comparar o conteúdo de objetos, especialmente Strings.

Ilustração da Memória e Referências para Strings

Strings criadas com `new String()` - Referências Diferentes

Strings Literais - Mesma Referência (String Pool)

Mesmo que s3 == s4 possa retornar true em certos casos devido a uma otimização do Java (o "String Pool"), para garantir que você está comparando o conteúdo textual e não a localização na memória, **sempre use .equals() para Strings**.

3.3. Valores Aceitos nas Condições

A expressão dentro dos parênteses de um if ou else if deve sempre resultar em um valor **booleano** (true ou false).

  • Variáveis booleanas: if (estaLogado)
  • Operadores relacionais: (<, >, <=, >=, ==, !=) para comparar números. Ex: if (idade >= 18)
  • Chamadas de método que retornam booleanos: Ex: if (minhaString.isEmpty())
  • Operadores lógicos: (&&, ||, !) para combinar múltiplas condições. Ex: if (saldo > 0 && clienteAtivo)
boolean estaAtivo = true;
int nivel = 5;

if (estaAtivo) { // Condição com booleano
    System.out.println("Usuário ativo.");
}

if (nivel > 3) { // Condição com operador relacional
    System.out.println("Nível alto.");
}

String nome = "Maria";
if (nome.equals("Maria") || nome.equals("Joana")) { // Condição com .equals() e operador lógico
    System.out.println("Nome comum.");
}

3.4. A Importância da Ordem das Condições no else if

Em uma cadeia de if-else if-else, as condições são avaliadas na ordem em que aparecem. A primeira condição que for verdadeira terá seu bloco executado, e as demais serão ignoradas.

Cuidado com a ordem: Coloque as condições mais específicas (ou restritivas) primeiro e as mais gerais por último.

Exemplo de ordem incorreta (e o porquê):

int score = 95;
if (score >= 70) {
    System.out.println("Aprovado"); // Esta condição pega 95, 80, 70...
} else if (score >= 90) { // NUNCA será alcançado se o score for >= 90, pois o primeiro IF já pegou!
    System.out.println("Parabéns!"); 
} else {
    System.out.println("Reprovado");
}
// Saída: "Aprovado" (o "Parabéns!" nunca aparece)

Exemplo de ordem correta:

int score = 95;
if (score >= 90) { // Condição mais específica primeiro
    System.out.println("Parabéns! (Conceito A)"); // Este será executado
} else if (score >= 70) { 
    System.out.println("Aprovado (Conceito C)");
} else {
    System.out.println("Reprovado");
}
// Saída: "Parabéns! (Conceito A)"

4. Na Prática: Exemplos de Código

Vamos ver como essas estruturas funcionam com exemplos reais.

Exemplo 1: Condição Simples com if

public class ExemploIf {
    public static void main(String[] args) {
        int temperatura = 25; // Define a temperatura

        if (temperatura > 20) { // Condição: Se temperatura for maior que 20...
            System.out.println("Está calor hoje!"); // ...então imprime esta mensagem
        }
        System.out.println("Fim do programa."); // Esta linha sempre executa
    }
}

Explicação: Aqui, a condição `temperatura > 20` é verdadeira (25 é maior que 20). Portanto, a mensagem "Está calor hoje\!" é impressa. A mensagem "Fim do programa." é impressa em ambos os casos, pois está fora do bloco `if`.

Exemplo 2: Uso de if e else

public class ExemploIfElse {
    public static void main(String[] args) {
        boolean temCafe = false; // Define se há café

        if (temCafe) { // Condição: Se temCafe for verdadeiro...
            System.out.println("Que bom, vou tomar café!"); // ...executa este bloco
        } else { // Senão (se temCafe for falso)...
            System.out.println("Preciso fazer mais café."); // ...executa este outro bloco
        }
    }
}

Explicação: Como `temCafe` é `false`, o bloco `if` é ignorado e o bloco `else` é executado, imprimindo "Preciso fazer mais café."

Exemplo 3: Uso de if, else if e else (Sistema de Notas)

import java.util.Scanner; // Importa a classe Scanner para ler entrada do usuário

public class ExemploNotas {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in); // Cria um objeto Scanner

        System.out.print("Digite a nota do aluno (0-100): ");
        int nota = scanner.nextInt(); // Lê a nota digitada pelo usuário

        if (nota >= 90) { // Primeira condição: Se nota for 90 ou mais...
            System.out.println("Excelente! Conceito A.");
        } else if (nota >= 70) { // Senão, se nota for 70 ou mais...
            System.out.println("Muito bom! Conceito B.");
        } else if (nota >= 50) { // Senão, se nota for 50 ou mais...
            System.out.println("Aprovado! Conceito C.");
        } else { // Senão (se nenhuma das anteriores for verdadeira)...
            System.out.println("Reprovado. Precisa estudar mais.");
        }
        
        scanner.close(); // Fecha o Scanner para liberar recursos
    }
}

Explicação: O programa pede uma nota. Ele testa as condições em cascata. Se a nota for 85, por exemplo, o primeiro `if` (>=90) é falso. O primeiro `else if` (>=70) é verdadeiro, então imprime "Muito bom\! Conceito B." e as demais condições são ignoradas.

Exemplo 4: Uso com Comparação de Strings (`==` vs `.equals()`)

public class ExemploStringComparacao {
    public static void main(String[] args) {
        String nome1 = "Java";
        String nome2 = "Java"; // Literal, pode estar no String Pool
        String nome3 = new String("Java"); // Cria um novo objeto String

        // Comparação com ==
        System.out.println("nome1 == nome2: " + (nome1 == nome2)); // Provavelmente true (String Pool)
        System.out.println("nome1 == nome3: " + (nome1 == nome3)); // False (referências diferentes)

        // Comparação com .equals()
        System.out.println("nome1.equals(nome2): " + (nome1.equals(nome2))); // True (conteúdo igual)
        System.out.println("nome1.equals(nome3): " + (nome1.equals(nome3))); // True (conteúdo igual)

        String senhaCorreta = "senai123";
        String senhaDigitada = "senai123"; // Simula entrada do usuário

        if (senhaDigitada.equals(senhaCorreta)) { // Forma CORRETA de comparar Strings
            System.out.println("Senha correta! Acesso permitido.");
        } else {
            System.out.println("Senha incorreta! Acesso negado.");
        }
    }
}

Explicação: Este exemplo demonstra a diferença crítica. `==` pode dar resultados inconsistentes com Strings. `equals()` sempre compara o conteúdo, sendo a maneira segura e correta de verificar igualdade de texto.

Exemplo 5: Aplicação Prática (Verificar Idade para Votar)

import java.util.Scanner;

public class Eleitor {
    public static void main(String[] args) {
        Scanner input = new Scanner(System.in);
        
        System.out.print("Digite sua idade: ");
        int idade = input.nextInt();

        if (idade < 16) {
            System.out.println("Você ainda não pode votar.");
        } else if (idade >= 18 && idade <= 70) { // Combina duas condições com AND
            System.out.println("Seu voto é obrigatório!");
        } else if (idade >= 16 || idade > 70) { // Condição para voto facultativo
            System.out.println("Seu voto é facultativo.");
        } else {
            System.out.println("Idade inválida."); // Para idades negativas, por exemplo
        }

        input.close();
    }
}

Explicação: Um exemplo real de como if-else if-else é usado para categorizar uma idade em diferentes grupos de eleitores, utilizando também operadores lógicos `&&` (AND) e `||` (OR).

6. Armadilhas Comuns: Evitando Erros de Iniciante

Aprender é errar e corrigir. Mas já que estamos aqui, vamos destacar alguns erros frequentes para você tentar evitar:

  • Esquecer ou Omitir as Chaves { }: Mesmo para um comando, a falta de chaves pode levar a bugs difíceis de encontrar quando o código for modificado.
  • Confundir Atribuição (=) com Comparação (==): 
    int x = 10;
if (x = 5) { // ERRO COMUM! Isso tenta atribuir 5 a x, e não é uma condição booleana.
    // ...
}
    Sempre use `==` para comparar.
  • Usar == para Comparar Strings: Vimos que isso compara referências, não conteúdo. Use .equals() ou .equalsIgnoreCase().
  • Ordem Incorreta das Condições no else if: Se uma condição mais geral vier antes de uma mais específica, a específica pode nunca ser alcançada (como no exemplo do score >= 70 antes do >= 90).
  • Colocar Ponto e Vírgula ; Logo Após o if: 
    if (condicao); { // ERRO! O ; "encerra" o if, o bloco seguinte é executado SEMPRE.
    System.out.println("Isso sempre vai aparecer.");
}
    O ponto e vírgula indica o fim de uma instrução. Um `if` não termina ali, ele espera um bloco de código.
