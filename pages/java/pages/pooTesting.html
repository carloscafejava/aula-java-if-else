Programação Orientada a Objetos em Java

Guia completo para dominar POO: Classes, Objetos, Encapsulamento, Herança e Polimorfismo

1. O Básico: Entendendo POO

Java é uma linguagem orientada a objetos e não é possível desenvolver nenhum software sem seguir o paradigma da orientação a objetos. Um sistema orientado a objetos é um conjunto de classes e objetos que interagem entre si, de modo a gerar um resultado esperado.

Pense em um bolo de chocolate. A receita possui características que determinam a receita como os ingredientes. Imaginemos a receita como sendo a Classe, e o bolo pronto como sendo o objeto. Os ingredientes são os atributos da Classe Receita. O modo de preparo do bolo é o método que vamos utilizar para elaborar o bolo.

O bolo pronto é único, pois se você fizer este mesmo bolo seguindo a mesma receita, provavelmente o seu bolo será diferente do meu. Estas características do bolo pertencem somente ao meu ou ao seu bolo, assim são atributos do objeto, também denominado de atributos de instância.

2. Classes e Objetos: Os Fundamentos

2.1. O que é uma Classe?

Uma classe é como uma receita ou um modelo que define as características (atributos) e comportamentos (métodos) de um objeto. Podemos criar uma classe PESSOA que possui atributos e métodos:

Estrutura Conceitual de uma Classe:
java 
PESSOA
    cor_de_olho
    cor_de_cabelo
    idade
    
    pinta_cor_cabelo()
    faz_aniversario()

2.2. Criando Objetos (Instanciando)

Ao instanciar uma classe PESSOA, obtemos o que chamamos de objeto:

java 
daniela = new PESSOA();
bosco = new PESSOA();

// Cada objeto tem suas características particulares
daniela.cor_de_olho = "verde";
daniela.cor_de_cabelo = "castanho escuro";
daniela.idade = 27;

bosco.cor_de_olho = "castanho";

Explicação:

  • new PESSOA() → Cria uma nova instância (objeto) da classe PESSOA
  • daniela e bosco → São variáveis que referenciam objetos diferentes da mesma classe
  • Operador ponto (.) → Usado para acessar atributos e métodos do objeto
  • Atributos → Cada objeto possui seus próprios valores

7. Modificadores de Acesso

Java fornece modificadores de acesso para controlar a visibilidade de atributos e métodos:

Modificador Descrição Visibilidade
private Mais restritivo Acessível apenas dentro da própria classe
protected Intermediário Acessível na própria classe e nas subclasses
package (default) Sem modificador Acessível apenas no mesmo pacote
public Menos restritivo Acessível de qualquer lugar

Sobre Protected:

Protected deixa os atributos da classe pai visíveis, acessíveis "hierarquia abaixo". Mas para o restante do programa tem o mesmo efeito que private.

Importante: "A herança permite que uma subclasse ganhe acesso a declarações protected de sua superclasse, mas o usuário não percebe isso, para o usuário (uma classe externa) o que continua existindo é o que é public".

📝 Exercício 2: Classe Carro com Métodos

Crie uma classe Carro com atributos: marca, modelo, ano e velocidadeAtual. Adicione métodos para acelerar (aumenta velocidade em 10), frear (diminui velocidade em 10, não pode ser negativa) e mostrar status.

Mostrar Solução
java 
public class Carro {
    private String marca;
    private String modelo;
    private int ano;
    private int velocidadeAtual;
    
    public Carro(String marca, String modelo, int ano) {
        this.marca = marca;
        this.modelo = modelo;

Crie uma classe Animal com atributos nome e idade. Depois crie duas subclasses: Cachorro e Gato. Cada uma deve ter um método emitirSom() que imprime o som característico do animal.

📝 Exercício 4: Conta Bancária Completa

Crie uma classe ContaBancaria com saldo privado, métodos depositar, sacar (com validação) e consultarSaldo. Adicione um construtor que inicialize com saldo zero.

Mostrar Solução
java 
public class ContaBancaria {
    private String titular;
    private double saldo;
    
    public ContaBancaria(String titular) {
        this.titular = titular;
        this.saldo = 0.0;
    }
    
    public void depositar(double valor) {
        if (valor > 0) {
            this.saldo += valor;
            System.out.println("Depósito de R$ " + valor + " realizado com sucesso!");
        } else {
            System.out.println("Valor inválido para depósito!");
        }
    }
    
    public void sacar(double valor) {
        if (valor > 0 && valor <= this.saldo) {
            this.saldo -= valor;
            System.out.println("Saque de R$ " + valor + " realizado com sucesso!");
        } else if (valor > this.saldo) {
            System.out.println("Saldo insuficiente!");
        } else {
            System.out.println("Valor inválido para saque!");
        }
    }
    
    public void consultarSaldo() {
        System.out.println("Titular: " + this.titular);
        System.out.println("Saldo atual: R$ " + this.saldo);
    }
    
    public double getSaldo() {
        return this.saldo;
    }
}

📝 Exercício 5: Polimorfismo - Formas Geométricas

Crie uma classe Forma com um método calcularArea(). Crie subclasses Circulo (com raio) e Retangulo (com largura e altura) que sobrescrevem o método calcularArea.

Mostrar Solução
java 
public class Forma {
    public double calcularArea() {
        return 0.0;
    }
}

public class Circulo extends Forma {
    private double raio;
    
    public Circulo(double raio) {
        this.raio = raio;
    }
    
    @Override
    public double calcularArea() {
        return Math.PI * Math.pow(this.raio, 2);
    }
}

public class Retangulo extends Forma {
    private double largura;
    private double altura;
    
    public Retangulo(double largura, double altura) {
        this.largura = largura;
        this.altura = altura;
    }
    
    @Override
    public double calcularArea() {
        return this.largura * this.altura;
    }
}

// Testando Polimorfismo
public class TesteFormas {
    public static void main(String[] args) {
        Forma[] formas = new Forma[2];
        formas[0] = new Circulo(5);
        formas[1] = new Retangulo(4, 6);
        
        for (Forma f : formas) {
            System.out.println("Área: " + f.calcularArea());
        }
    }
}

9. Resumo dos Conceitos de POO

🎓 Pilares da Programação Orientada a Objetos:

  • Encapsulamento: Ocultar informações e proteger dados usando modificadores de acesso (private, protected, public)
  • Herança: Reutilizar código através da palavra-chave extends, criando hierarquias de classes
  • Polimorfismo: Objetos de diferentes classes podem responder ao mesmo método de formas diferentes
  • Abstração: Focar nas características essenciais, escondendo detalhes de implementação

🔑 Conceitos Importantes:

  • Classe: Modelo ou receita que define atributos e métodos
  • Objeto: Instância de uma classe criada com new
  • Atributos: Características/variáveis que definem o estado do objeto
  • Métodos: Comportamentos/ações que o objeto pode executar
  • Construtor: Método especial chamado ao criar um objeto
  • this: Referência ao objeto atual
  • super: Referência à classe pai
  • static: Pertence à classe, não aos objetos individuais

💡 Boas Práticas:

  • Sempre use private para atributos e forneça métodos públicos (getters/setters) quando necessário
  • Não crie getters e setters desnecessários - só crie quando realmente precisar
  • Use construtores para garantir que objetos sejam criados em estado válido
  • Prefira composição sobre herança quando apropriado
  • Mantenha alta coesão (responsabilidade única) e baixo acoplamento entre classes
  • Use nomes descritivos para classes (substantivos), métodos (verbos) e atributos

5. Herança em Java

A herança é um mecanismo que define variáveis e métodos comuns a todos os objetos de um certo tipo. A partir de uma determinada classe, pode-se ter subclasses e superclasses.

5.1. Exemplo: Hierarquia de Funcionários

java 
class EmpregadoDaFaculdade {
    private String nome;
    private double salario;

    double getGastos() {
        return this.salario;
    }

    String getInfo() {
        return "nome: " + this.nome + " com salário " + this.salario;
    }
}

5.2. Criando uma Subclasse

java 
class ProfessorDaFaculdade extends EmpregadoDaFaculdade {
    private int horasDeAula;

    double getGastos() {
        return this.getSalario() + this.horasDeAula * 10;
    }

    String getInfo() {
        String informacaoBasica = super.getInfo();
        String informacao = informacaoBasica + " horas de aula: " +
                            this.horasDeAula;
        return informacao;
    }
}

Explicação da Herança:

  • extends → Indica que ProfessorDaFaculdade herda de EmpregadoDaFaculdade
  • Reescrita de métodos → A classe filha pode sobrescrever métodos da classe mãe
  • super.getInfo() → Chama o método da classe mãe para reaproveitar código
  • Novos atributos → A classe filha pode ter seus próprios atributos (horasDeAula)

5.3. Exemplo com Hierarquia de Pontos

java 
//Classe Ponto
class Ponto {
    protected float x, y;

    public Ponto(float ax, float ay) {
        this.x = ax;
        this.y = ay;
    }

    public void mostra() {
        System.out.println("(" + this.x + "," + this.y + ")");
    }
}

//Classe PtoMove
class PtoMove extends Ponto {
    
    public PtoMove(float a, float b) {
        super(a, b); // chamada do construtor da classe pai
    }

    public void move(float dx, float dy) {
        this.x += dx;
        this.y += dy;
    }
}

//Classe PtoReflete
class PtoReflete extends Ponto {
    
    public PtoReflete(float a, float b) {
        super(a, b);
    }

    void reflete() {
        this.x = -this.x;
        this.y = -this.y;
    }
}

6. Polimorfismo

Polimorfismo é quando duas ou mais classes derivadas de uma mesma superclasse podem invocar métodos que possuem a mesma identificação (assinatura), mas comportamentos distintos.

java 
class GeradorDeRelatorio {
    public void adiciona(EmpregadoDaFaculdade f) {
        System.out.println(f.getInfo());
        System.out.println(f.getGastos());
    }
}

// Podemos passar qualquer tipo de EmpregadoDaFaculdade
GeradorDeRelatorio relatorio = new GeradorDeRelatorio();
relatorio.adiciona(new ProfessorDaFaculdade());
relatorio.adiciona(new EmpregadoDaFaculdade());

✨ A Magia do Polimorfismo:

Podemos passar para a classe GeradorDeRelatorio qualquer EmpregadoDaFaculdade! Vai funcionar tanto para professor, quanto para funcionário comum.

Quem programou a classe GeradorDeRelatorio nunca imaginou que existiria uma classe Reitor, e mesmo assim o sistema funciona quando criamos essa nova classe depois.

        System.out.println("Saldo atual: R$ " + this.saldo);
    }
    
    public double getSaldo() {

📝 Exercício 5: Polimorfismo - Formas Geométricas

Crie uma classe Forma com um método calcularArea(). Crie subclasses Circulo (com raio) e Retangulo (com largura e altura) que sobrescrevem o método calcularArea.

Mostrar Solução
java 
public class Forma {
    public double calcularArea() {
        return 0.0;
    }
}

public class Circulo extends Forma {
    private double raio;
    
    public Circulo(double raio) {
        this.raio = raio;
    }
    
    @Override
    public double calcularArea() {
        return Math.PI * Math.pow(this.raio, 2);
    }
}

public class Retangulo extends Forma {
    private double largura;
    private double altura;
    
    public Retangulo(double largura, double altura) {
        this.largura = largura;
        this.altura = altura;
    }
    
    @Override
    public double calcularArea() {
        return this.largura * this.altura;
    }
}

// Testando Polimorfismo
public class TesteFormas {
    public static void main(String[] args) {
        Forma[] formas = new Forma[2];
        formas[0] = new Circulo(5);
        formas[1] = new Retangulo(4, 6);
        
        for (Forma f : formas) {
            System.out.println("Área: " + f.calcularArea());
        }
    }
}

9. Resumo dos Conceitos de POO

🎓 Pilares da Programação Orientada a Objetos:

  • Encapsulamento: Ocultar informações e proteger dados usando modificadores de acesso (private, protected, public)
  • Herança: Reutilizar código através da palavra-chave extends, criando hierarquias de classes
  • Polimorfismo: Objetos de diferentes classes podem responder ao mesmo método de formas diferentes
  • Abstração: Focar nas características essenciais, escondendo detalhes de implementação

🔑 Conceitos Importantes:

  • Classe: Modelo ou receita que define atributos e métodos
  • Objeto: Instância de uma classe criada com new
  • Atributos: Características/variáveis que definem o estado do objeto
  • Métodos: Comportamentos/ações que o objeto pode executar
  • Construtor: Método especial chamado ao criar um objeto
  • this: Referência ao objeto atual
  • super: Referência à classe pai
  • static: Pertence à classe, não aos objetos individuais

💡 Boas Práticas:

  • Sempre use private para atributos e forneça métodos públicos (getters/setters) quando necessário
  • Não crie getters e setters desnecessários - só crie quando realmente precisar
  • Use construtores para garantir que objetos sejam criados em estado válido
  • Prefira composição sobre herança quando apropriado
  • Mantenha alta coesão (responsabilidade única) e baixo acoplamento entre classes
  • Use nomes descritivos para classes (substantivos), métodos (verbos) e atributos
